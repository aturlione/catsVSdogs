Utilizando convnets con datasets pequeños¶

Lo primero que tienes que hacer es descargarte el dataset de https://lara.web.cern.ch/lara/train.zip en la terminal de Jupyter y descomprimirlo en la misma carpeta donde se encuentra esta libreta.

Para descargar otro conjunto de datos desde imagenet se puede descargar la lista con las URL a las imgenes y usar wget -i

Entrenando desde 0 una convNet¶

Entrenar un modelo de clasificacin de imgenes con muy pocos datos es una situacin comn en la que te encontrars si acabas dedicndote a hacer Computer Vision en un contexto profesional.

Tener "pocas" muestras puede significar cualquier cosa entre unos pocos cientos y unas pocas decenas de miles de imgenes. Vamos a ilustrar aqui un ejemplo prctico: vamos a centrarnos en clasificar imgenes como "perros" y "gatos".

La relevancia del Deep Learning en problemas con pocos datos¶

Quizs habrs oido muchas veces que el Deep Learning solo funciona cuando se tienen grandes cantidades de datos. Esto en parte es verdad: una de las caractersticas del Deep learning es que puede encontrar caractersticas interesantes a partir del dataset de entrenamiento por si mismo, y esto a priori es ms sencillo cuando se tienen muchos ejemplos disponibles, especialmente en el caso de tener datasets de input con una alta dimensionalidad, como es el caso de las imgenes.

Sin embargo, lo que constituye un dataset "grande" es relativo. Concretamente relativo al tamao y la profundidad de la red que estamos intentando entrenar. No es posible enrenar una convnet para que resulta un problema completo con solo unas decenas de ejemplos, pero unos pocos cientos puede ser suficiente si el modelo est bien montado (entenderemos que significa bien "montado" a lo largo del curso de Deep Learning).

Como las convnets aprenden caractersticas locales, invariantes bajo translaciones, son muy eficientes en cuanto al nmero de imgenes necesarias para llevar a cabo problemas perceptuales. As que entrenar una convnet desde 0 con un dataset no muy grande an nos puede llevar a resultados razonables como veremos aqui.

Pero hay ms an: los modelos de Deep Learning son altamente "reciclables". Uno puede coger, por ejemplo, un problema de clasificacin de imagen y un convertidor de voz a texto entrenado sobre un dataset muy grande y luego reutilizarlo para resolver otro problema completamente distinto solo aadindole pequeas modificaciones. Ms especificamente, en el caso de Computer Vision, muchos modelos pre-entrenados (normalmente entrenados en el dataset ImageNet) son hechos publicos para que uno pueda descargarlos y utilizarlos para crear potentes modelos de Computer Vision con muy pocos datos.

Pero aqui nos vamos a limitar a correr un ejemplo sencillito.

Construyendo nuestra red¶

En el anterior ejemplo hemos construido una pequea convnet para resolver el problema de clasificar nmeros escritos a mano usando el dataset MNIST, as que ya estamos familiarizados con la terminologa que utiliza keras. Vamos a reutilizar la estructura general que teniamos en el ejemplo anterior: nuestra convnet tendra una pila de capas alternadas de Conv2D (con activacin relu ) y capas MaxPooling2D.

Sin embargo, como estamos tratando con imgenes mayores y un problema ms complejo, vamos a crear nuestra red en consecuencia: tendr una capa ms de Conv2D + MaxPooling2D. Esto sirve para aumentar la capacidad de la red y para reducir an ms el tamao de los mapas de caractersticas, para que no sean tan enormes cuando lleguen al paso de aplanado. Empezamos usando imgenes de input de 150x150 (una eleccin arbitraria), y acabaremos con mapas de caractersticas que tienen un tamao de 7x7 antes de la capa de aplanamiento.

Es importante tener en cuenta que la profundidad de los mapas de caractersticas va creciendo progresivamente segn se avanza en la red neuronales ( de 32 a 128) mientras que el tamao de los mapas de caractersticas va disminuyendo (de 148x148 a 7x7). Este patrn lo vers en casi todas las convnets.

Como estamos atacando un problema de clasificacin binaria (perro o gato), vamos a acabar la red con una nica unidad (una capa densa de tamao 1) y con una activacin sigmoide. Esta unidad codificar la probabilidad de que nuestra red est mirando a una clase o a otra.

El aspecto final del modelo debe de ser el siguiente:

Preprocesado de datos¶

Las imgenes deben estar formateadas apropiadamente en tensores de flotantes antes de drselas a la red. Esto es justo lo que vamos a hacer aqui. Antes de preprocesarlas las imgenes son archivos JPEG. Los pasos para poder darlos a nuestra red son a grandes rasgos:

  • Leer los archivos con las imgenes.
  • Decodificar el contenido del JPEG en una "parrilla" con el RGB de los pixels
  • Convertir esa "parrilla" en tensores de flotantes
  • Re-escalar los valores de los pixels (enre 0 y 255) al intervalo [0, 1] ya que las redes neuronales prefieren trabajar con valores pequeos.

Todo esto puede parecer muy complicado pero gracias a Keras nuestra vida es mucho ms fcil y podemos contar con tus herramientas para ocuparse de estos pasos automaticamente. Keras tiene un mdulo con herramientas para el tratamiento de imgenes, que se puede encontrar en keras.preprocessing.image. En particular, contiene la clase ImageDataGenerator que nos permite automaticamente convertir imgenes que tengamos en el disco duro en tensores pre-procesados. Esto es justamente lo que usaremos a continuacin. Para ello podemos utilizar el flow_from_directory para coger las imgenes directamente de las carpetas que generamos previamente. Le damos como entrada las carpetas donde estn las imgenes de entrenamiento (o validacin), el tamao de las imgenes (target_size), tamao del batch que vamos a usar (vamos a empezar por 20) y como solo hay dos categoras, le decimos que vamos a usar binary_crossentropy (class_mode). Al correr estos comandos obtendremos lo siguiente el nmero total de imgenes y cuantas clases hay.
